Network security (SSRF protection)

By default, Nexus refuses to call private, loopback, or internal addresses — this blocks a provider URL from being pointed at cloud metadata or your internal network. Running a local model (Ollama, LM Studio)? Add its host below to allow just that one.

Content guardrails

Optional prompt/response filtering — redact PII or block banned content and injection patterns. Off by default. Rules are a JSON array; each has a name, pattern (regex), action (block or redact), and optional appliesTo (input/output/both) and replacement.

Presets you can copy: email, us-phone, credit-card, ssn, api-key, prompt-injection.
Cost-aware routing

Among providers that are healthy and in-headroom within a tier, bias routing toward the cheaper one (using model registry pricing). This only breaks ties — it never overrides tier priority, circuit-breaker/rate-limit state, or sticky cache affinity.

Response cache

Cache exact-match responses (same model, messages, and params). A repeat request is served straight from Redis — a real $0 call, the provider is skipped entirely. Off by default. Identical requests return the cached answer until the TTL expires.

Notifications

Get alerted when the gateway degrades or is attacked, so you don't have to watch this dashboard. Off by default. Sends at most one message per event per window (a flapping key won't flood you). Email via Resend (free tier), and/or a generic webhook (Slack/Discord/PagerDuty). Delivery never touches the request path.
